## 1. โจทย์ของทีม

**Problem statement** (1-2 ประโยค บอกว่าจะแก้ปัญหาอะไรในชีวิตจริง):

```
เราจะสร้างโซลูชั่นในการแยกขยะให้ง่ายและสะดวกข้นไม่ต้องคอยมาแยกเอง
```

---

## 2. Class Definition

ระบุ class ที่จะใช้ + เกณฑ์ที่ทำให้แต่ละ class แตกต่าง:

| # | ชื่อ Class | คำอธิบาย (3-4 ประโยค) | สิ่งที่ทำให้ต่างจาก class อื่น |
|---|---|---|---|
| 1 | กล่องพลาสติก | มีรูปทรงสี่เหลี่ยม สีใส | รูปทรง |
| 2 | กระบอกน้ำ | ทรงกระบอกสีฟ้า | สี |
| 3 | ขวดน้ำ | ทรงกระบอกสีใส | มองทะลุได้ |

---

## 3. Edge Cases — สิ่งที่อาจสับสน

ระบุสถานการณ์ที่อาจทำให้โมเดลสับสน:

```
1. _________________________________________________
   "กล่องมีสีใสเหมือนกัน" อาจถูกจัดเป็น "ขวดน้ำ"

2. _________________________________________________
   "กระบอกน้ำและขวดน้ำมีรูปทรงเหมือนกัน" อาจถูกจัดเป็น "ขวดน้ำ"
3. _________________________________________________
   "ขวดน้ำกล่องมีสีใสเหมือนกัน" อาจถูกจัดเป็น "ขวดน้ำ"
```

**วางแผนแก้:**
```
[เก็บดาต้าให้ครบทุกมุมและทุกสภาพแวดล้อมให้ได้มากที่สุด]
```

## 5. Diversity Plan — กันอคติ (Bias)

ตอบ 3 คำถาม:

### a) มีหลาย "environment" ไหม?
```
[เก็บในห้องเรียน + ห้องอื่น? แสงสว่าง/แสงน้อย? เวลาต่างกัน?]
```

### b) มีหลาย "subject" ไหม? (กรณี Vision/Audio/Motion)
```
[เก็บจากหลายคนในทีม? ลักษณะ background ต่างกัน?]
```

### c) มีหลาย "variation" ไหม?
```
[มุม/ระยะ/ความเร็ว/ระดับเสียง ที่แตกต่างกัน?]
```

