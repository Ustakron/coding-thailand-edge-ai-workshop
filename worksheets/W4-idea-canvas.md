## 🎯 1. Problem & User

### a) เราจะแก้ปัญหาอะไร?
```
[ผู้พิการทางการได้ยินหรือผู้ที่สื่อสารด้วยภาษามืออาจสื่อสารกับคนทั่วไปได้ยากเพราะหลายคนอ่านภาษามือไม่เข้าใจโปรเจกต์นี้ช่วยแปลท่าภาษามือพื้นฐานให้กลายเป็นเสียงพูดเพื่อให้สื่อสารกันได้ง่ายขึ้นในชีวิตประจำวัน]
```

### b) ใครคือ user หลัก?
```
[เขียนกลุ่มเป้าหมาย — ระบุให้ชัด ไม่ใช่ "ทุกคน"]

- ผู้พิการทางการได้ยินที่ใช้ภาษามือในการสื่อสาร
- ครู นักเรียน หรือเพื่อนร่วมชั้นที่ต้องการสื่อสารกับผู้ใช้ภาษามือ
- หน่วยบริการ/จุดประชาสัมพันธ์ที่ต้องสื่อสารกับคนหลากหลายกลุ่ม
```

### c) ทำไม Edge AI ถึงเหมาะกับปัญหานี้? (ทำไมไม่ใช่ Cloud)
```
Edge AI เหมาะเพราะต้องตอบสนองเร็วเมื่อผู้ใช้ทำท่าภาษามือไม่ต้องรอส่งข้อมูลขึ้น Cloud ช่วยรักษาความเป็นส่วนตัวเพราะภาพมือไม่ต้องส่งออกไปอินเทอร์เน็ตและสามารถใช้งานได้แม้ในพื้นที่ที่เน็ตไม่เสถียร
```

## 💎 2. Value Proposition

### Unique Value (USP)
ในประโยคเดียว:

```
"เสียงที่ไม่ได้ออกจากปาก" ช่วยผู้ใช้ภาษามือให้สื่อสารกับคนทั่วไปผ่านเสียงพูดได้ทันทีต่างจากแอปแปลภาษาทั่วไป
เพราะทำงานบนอุปกรณ์ Edge AI ตอบสนองเร็วและไม่จำเป็นต้องใช้อินเทอร์เน็ตตลอดเวลา
```


## 🛠️ 3. Solution

### a) Hardware ที่จะใช้
- UNO Q + Modulino พื้นฐาน
- เพิ่มเติม:
  - Modulino: Pixels / Buttons
  - Webcam : สำหรับเก็บภาพ
  - Speaker / Buzzer : สำหรับส่งเสียง

### b) AI Model
Track: Vision 
Classes: สวัสดี, ขอบคุณ, ช่วยด้วย, ใช่, ไม่ใช่

จะ improve จาก V2 ตอนนี้อย่างไร?

เปลี่ยนข้อมูลเป็นวิดีโอของแต่ละท่าให้หลากหลาย เช่น มุมกล้อง แสง ฉากหลัง และขนาดมือ
ปรับ label ให้ชัดเจน ลดภาพที่เบลอหรือท่าที่ไม่ตรง
และทดสอบโมเดลกับคนในทีมหลายคนเพื่อให้ระบบจำท่าได้แม่นขึ้น

### c) Form Factor
- จะติดที่ไหน? ใช้ยังไง?
  ```
  เป็นกล่องต้นแบบขนาดเล็กมี กล้อง/บอร์ด UNO Q สำหรับมองท่ามือ
  ผู้ใช้ยกมือทำภาษามือหน้ากล้องระบบจะประมวลผลบนบอร์ด
  จากนั้นจึงส่งเสียงคำแปลออกลำโพง
  ```

## 💡 Tips สำหรับช่วงต่อยอด

1. **ถ้าต้องอธิบายงาน 2 นาที ทีมควรแบ่งช่วงพูดกันล่วงหน้า**
2. **ถ้าต้องโชว์สด ควรมีแผนสำรองเมื่อ internet หรือ hardware มีปัญหา**
3. **Document ทุก iteration** ใน GitHub — commit ทุก hour
4. **Token = constraint ที่สอนการตัดสินใจ** — เลือกสิ่งจำเป็น
5. **ไอเดียดี + execute ไม่จบ < ไอเดียเรียบง่าย + execute เสร็จ**
